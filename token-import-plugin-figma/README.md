# Variables Tokens Collections Importer — Figma Plugin [v1.1 STABLE]

Import and export Figma Variables ZIPs generated by the Figma Variables Tokens Generator skill.

## Installation
- **Figma Community (Recommended)**: [Install Variables Tokens Collections Importer](https://www.figma.com/community/plugin/1619733963699677957)
- **Local Development**: 
  1. Open Figma Desktop
  2. Menu → Plugins → Development → Import plugin from manifest
  3. Select the `manifest.json` from this folder
  4. The plugin will appear under Plugins → Development → Token Porter

## Structural Sync (Source of Truth)

**Variables Tokens Collections Importer** treats your ZIP archive as the absolute source of truth. If a collection exists in Figma:
- **Replace**: Synchronizes the existing Figma collection to exactly match the ZIP. It adds missing tokens/modes, updates existing values, and **deletes** any tokens or modes that are not present in the ZIP.
- **Cancel**: Aborts the import and returns to the file selection screen.

## Metadata & Scopes

The plugin now supports advanced Figma metadata:
- **Scopes**: Automatically maps design token categories to Figma's internal scope enums (e.g., `STROKE_COLOR`, `FRAME_FILL`).
- **Code Syntax**: Preserves and applies platform-specific syntax (Web, Android, iOS).
- **Visibility**: Supports "Hidden from Publishing" settings.

## AI Skill Support

This plugin is designed to work seamlessly with the **Figma Variable Creator** AI skill.
1. Use the skill to generate your variables structure.
2. Ask the skill to "Export as Token Porter ZIP".
3. Drop the resulting ZIP here.

## ZIP Structure (Import format)

```
design-tokens.zip
├── 1. Primitives/
│   └── primitives.tokens.json
├── 2. Theme/
│   ├── light.tokens.json
│   └── dark.tokens.json
...
```


## Notes

- Requires internet connection on first use (loads JSZip from CDN).
- **Corrupt Token Protection**: New variables with unresolved aliases are automatically pruned to keep your file clean.
- **Pro Tip**: If variable values aren't updating, ensure your Alias paths are exactly correct in the JSON.
